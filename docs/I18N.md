# Internationalization (i18n) System

This project uses **react-i18next** for internationalization with a strongly-typed, cascading translation structure.

## Overview

The i18n system is designed to:
- Provide type-safe translations across all supported languages (PT, EN, ES)
- Enforce complete translations at build time
- Use a cascading/hierarchical structure for better organization
- Support dynamic content like project descriptions

## Structure

### Translation Files

Translation files are located in `src/i18n/locales/`:
- `pt.ts` - Portuguese (Brazil) - Source of truth for structure
- `en.ts` - English (US)
- `es.ts` - Spanish

## Usage in Components

### Import the hook

```tsx
import { useI18n } from '../hooks/useLanguage';

export const MyComponent = () => {
    const { t, language, setLanguage } = useI18n();
    
    // ...
};
```

### Use translations

```tsx
// Simple translation
<h1>{t('hero.info.title.greeting')}</h1>

// Dynamic translation with project ID
<h2>{t(`projects.items.${project.id}.title`)}</h2>

// Array translations
{(t(`projects.items.${project.id}.highlights`, { returnObjects: true }) as string[]).map(...)}

// Change language
<button onClick={() => setLanguage('en')}>English</button>
```

## Type Safety

The system enforces type safety in two ways:

1. **Structure Validation**: All translation files must have the same structure as `pt.ts`
2. **Build-Time Checks**: Missing translations will cause TypeScript compilation errors

### Adding New Translations

When adding a new translation key:

1. Add it to `src/i18n/locales/pt.ts` (source of truth)
2. Add the same key with translated text to `en.ts` and `es.ts`
3. Run `npm run build` to verify all translations are present

If you forget to add a translation to any language, the build will fail with a TypeScript error.

## Project-Specific Content

Project content (titles, descriptions, etc.) is stored in the translation files under `projects.items.{project-id}`. The `projects.ts` file only contains:
- `id` - Unique identifier (used as translation key)
- `image` - Project image path
- `tags` - Technology tags
- `type` - 'professional' or 'study'
- `stack` - Full technology stack
- `github` - Optional GitHub repository URL

## Configuration

i18next is configured in `src/i18n/config.ts` with:
- Default language: Portuguese (pt)
- Fallback language: Portuguese (pt)
- Language persistence in localStorage
- React Suspense disabled for compatibility

## Benefits

1. **Type Safety**: Missing or incorrect translations are caught at build time
2. **Organization**: Cascading structure makes translations easier to manage
3. **Maintainability**: Single source of truth for translation structure
4. **Scalability**: Easy to add new languages or translation keys
5. **Performance**: Efficient translation loading and caching
6. **DX**: Clear structure and autocomplete support in IDEs
